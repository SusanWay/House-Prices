# 🧩 Анализ категориальных признаков (Categorical Features)

## 📊 Что делает анализ

Функция анализирует строковые / категориальные признаки и помогает
понять:

-   есть ли пропуски
-   сколько уникальных значений
-   есть ли перекос категорий
-   есть ли редкие значения
-   как кодировать признак

------------------------------------------------------------------------

## 🧠 Описание метрик

### feature

Название признака.

------------------------------------------------------------------------

### missing

Есть ли пропуски (True / False).

------------------------------------------------------------------------

### missing\_%

Доля пропусков:

missing\_% = (кол-во NaN) / (общее количество строк)

📌 Интерпретация: 
- < 5% → нормально 
- 5--50% → требует анализа 
- 80% → чаще всего означает отсутствие объекта (например, нет бассейна)

------------------------------------------------------------------------

### unique_count

Количество уникальных значений.

📌 Интерпретация:
- 2 → бинарный признак
- 3--10 → обычная категория
- 20 → высокая кардинальность

------------------------------------------------------------------------

### unique\_%

Доля уникальных значений:

unique\_% = unique_count / total_count

📌 Если близко к 1 → возможно id или шум

------------------------------------------------------------------------

### top_value

Самое частое значение.

------------------------------------------------------------------------

### top_value\_%

Доля самого частого значения.

📌 Интерпретация:
- < 30% → норм
- 30--70% → перекос 
- 90% → почти константа (можно удалить)

------------------------------------------------------------------------

### rare_count

Количество категорий, встречающихся 1 раз.

------------------------------------------------------------------------

### rare\_%

Доля редких категорий:

rare\_% = rare_count / unique_count

📌 Интерпретация:
- 20% → нормально
- 50% → много шума

------------------------------------------------------------------------

### is_binary

True → признак содержит 2 значения

📌 Можно кодировать в 0/1

------------------------------------------------------------------------

### is_high_cardinality

True → много уникальных значений (\>20)

📌 Возможные решения:
- one-hot (если мало данных)
- target encoding 
- tree-based модели

------------------------------------------------------------------------

## 📌 Основные правила обработки

### Пропуски

-   Если \>80% → заменить на "None"
-   Если мало → заменить на mode

------------------------------------------------------------------------

### Категории

-   Небольшое количество → one-hot encoding
-   Есть порядок → ordinal encoding
-   Много значений → подумать про target encoding

------------------------------------------------------------------------

### Редкие категории

-   Объединить в "Other"

------------------------------------------------------------------------

### Почти константы

-   Если top_value\_% \> 90% → можно удалить

------------------------------------------------------------------------

## Итог

Анализ позволяет:

-   понять структуру категориальных данных
-   выбрать способ кодирования
-   очистить шум
-   подготовить данные для модели

Это ключевой этап перед обучением модели.
