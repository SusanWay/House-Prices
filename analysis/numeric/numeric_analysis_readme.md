# 📊 Анализ числовых признаков (Numeric Features)

## 📊 Что делает анализ

Функции позволяют анализировать числовые признаки и находить:

-   пропуски
-   выбросы (outliers)
-   асимметрию распределения (skew)
-   базовые статистики

------------------------------------------------------------------------

## 🧠 Метод определения выбросов

Используется метод **IQR (Interquartile Range)**.

Формулы:

Q1 = 25% квантиль\
Q3 = 75% квантиль\
IQR = Q3 - Q1

Границы:

lower = Q1 - 1.5 \* IQR\
upper = Q3 + 1.5 \* IQR

Значение считается выбросом, если:

x \< lower или x \> upper

------------------------------------------------------------------------

## 🔍 Описание функций

### get_outlier_mask

Возвращает маску выбросов (True / False для каждого значения).

------------------------------------------------------------------------

### detect_outliers

Проверяет, есть ли выбросы в колонке.

Возвращает: - True → есть выбросы - False → нет

------------------------------------------------------------------------

### get_outlier_percent

Возвращает долю выбросов:

outlier\_% = (кол-во выбросов) / (общее количество значений)

------------------------------------------------------------------------

### build_numeric_summary

Строит итоговую таблицу по всем числовым признакам.

------------------------------------------------------------------------

## 📊 Описание метрик

### feature

Название признака.

------------------------------------------------------------------------

### mean

Среднее значение:

mean = (x1 + x2 + ... + xn) / n

------------------------------------------------------------------------

### median

Медиана --- центральное значение после сортировки.

------------------------------------------------------------------------

### std

Стандартное отклонение --- мера разброса данных.

------------------------------------------------------------------------

### min / max

Минимальное и максимальное значение.

------------------------------------------------------------------------

### missing

Есть ли пропуски (True / False).

------------------------------------------------------------------------

### missing\_%

Доля пропусков:

missing\_% = NaN / общее количество

------------------------------------------------------------------------

### has_outliers

Есть ли выбросы (True / False).

------------------------------------------------------------------------

### outlier\_%

Доля выбросов:

outlier\_% = выбросы / общее количество

📌 Интерпретация:
- 1% → можно игнорировать
- 1--5% → допустимо
- 5--10% → стоит обработать
- 10% → обязательно обработать

------------------------------------------------------------------------

### skew

Асимметрия распределения.

📌 Интерпретация: 
- ≈ 0 → нормальное распределение
- 1 → сильный правый хвост
- -1 → сильный левый хвост

------------------------------------------------------------------------

## 📌 Основные правила обработки

### Пропуски

-   сильный skew → median
-   нормальное распределение → mean

------------------------------------------------------------------------

### Выбросы

-   не удаляем строки
-   используем clip:

x = min(x, Q_0.99)

------------------------------------------------------------------------

### Сильный skew

-   применяем log transform:

x' = log(1 + x)

------------------------------------------------------------------------

## Итог

Анализ числовых признаков позволяет:

-   понять распределение данных
-   найти выбросы
-   выбрать стратегию очистки
-   подготовить данные к обучению модели

Это ключевой этап в построении ML pipeline.
